## Noxaudit v1.1.6 is live

This release focuses on tightening security auditing logic and modernizing our documentation infrastructure.

### High-Severity Finding: Missing Tests
In [PR #129](https://github.com/atriumn/noxaudit/pull/129), we’ve introduced a new rule that flags repositories without a test suite as a **high-severity finding**. If your CI/CD pipeline relies on Noxaudit to gate releases, ensure your repositories contain valid test configurations to avoid audit failures.

### Documentation Engine Migration
We have migrated our documentation platform from MkDocs Material to **Fumadocs** ([PR #128](https://github.com/atriumn/noxaudit/pull/128)). This move improves the search experience and provides a more streamlined interface for navigating our CLI documentation.

### Summary of Changes
- **Security Audit Logic:** Repositories lacking tests are now marked high severity.
- **DX/Docs:** Migrated documentation site to Fumadocs for better performance and readability.
- **Versioning:** Cleanup from the v1.1.5 release cycle ([PR #118](https://github.com/atriumn/noxaudit/pull/118)).

To update your local installation, run:
`npm install -g noxaudit@latest` or update your `package.json` to `^1.1.6`.

Detailed release notes are available on [GitHub](https://github.com/atriumn/noxaudit/releases).
