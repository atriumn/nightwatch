## Noxaudit v1.2.4 is live

This release focuses on optimizing token usage and cleaning up CI/CD configuration to streamline the auditing workflow.

### LLM Routing Optimization
- **GPT-5-Mini for Pre-Passes:** In PR #151, we updated the scanning logic to route initial file pre-passes to `gpt-5-mini`. This avoids triggering long-context SKU pricing unnecessarily, reducing operational costs for large-scale codebase audits while maintaining the same level of security analysis.

### Infrastructure & CI
- **Dependabot Cleanup:** We have removed the invalid `npm_and_yarn` dependabot entry in PR #153. As the repository does not contain a `package.json`, this resolves persistent CI noise and weekly failures that were inherited from previous configuration templates.
- **Version Alignment:** v1.2.4 consolidates the fixes and chores introduced in the 1.2.3 release cycle (PR #149).

### Getting Started
To update your local environment to the latest version, pull the latest binary or update via your package manager:

```bash
noxaudit --version # 1.2.4
```

For a detailed diff of all changes, you can view the full [release notes](https://github.com/atriumn/noxaudit/releases/tag/v1.2.3).
