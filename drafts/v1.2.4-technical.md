## Noxaudit v1.2.4 Release

Version 1.2.4 is now available, bringing optimizations to the scan pipeline designed to reduce latency and overhead for large-scale code audits.

### GPT-5-Mini Pre-pass
The most significant change in this release is the migration of the initial scan pre-pass to `gpt-5-mini` ([PR #151](https://github.com/atriumn/noxaudit/pull/151)). By routing the preliminary analysis away from high-context SKUs, we’ve improved the efficiency of the audit lifecycle. This ensures that long-context models are only invoked for the deep-dive analysis where they are most effective.

### Release Details
- **v1.2.4 Stabilization:** Following the v1.2.3 tag ([PR #149](https://github.com/atriumn/noxaudit/pull/149)), this release hardens the release automation and CLI stability.
- **CI/CD Impact:** If you rely on automated audits, this update will automatically adjust your model routing to optimize for speed and cost.

### How to Update
Update your local CLI via npm:

- `npm install -g @atriumn/noxaudit@latest`

For full details on recent internal changes, view the [v1.2.3 release notes](https://github.com/atriumn/noxaudit/releases/tag/v1.2.3).

Thanks for using Noxaudit to secure your codebase.
