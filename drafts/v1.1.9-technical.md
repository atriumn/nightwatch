## Noxaudit 1.1.9: Hardening Batch Polling and Provider Interfaces

This release focuses on improving the robustness of the audit engine and refining our core abstractions for custom integrations.

### Core Interface Updates
In [PR #135](https://github.com/atriumn/noxaudit/pull/135), we added `retrieve_batch` to the `BaseProvider` interface. If you are maintaining internal providers for your codebase, ensure you implement this method to support optimized data fetching and avoid breaking changes in your local audit pipelines.

### Reliability and Error Handling
We have addressed a stability issue where the CLI would crash during batch polling under specific network conditions. [PR #134](https://github.com/atriumn/noxaudit/pull/134) introduces automatic retries for transient API errors, making the audit process significantly more resilient during large-scale scans.

### Release Highlights
- **Interface Refinement:** Added `retrieve_batch` to `BaseProvider` for better batch handling ([#135](https://github.com/atriumn/noxaudit/pull/135)).
- **Resiliency:** Fixed batch poll crashes by implementing retries for transient failures ([#134](https://github.com/atriumn/noxaudit/pull/134)).
- **Maintenance:** Routine release automation and version bumps to 1.1.8/1.1.9 ([#132](https://github.com/atriumn/noxaudit/pull/132)).

Check out the full [v1.1.8 release notes](https://github.com/atriumn/noxaudit/releases/tag/v1.1.8) for more context on the transition to the current v1.1.9 state.
