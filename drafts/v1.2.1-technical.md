## Noxaudit v1.2.1 is Live

Following the major v1.2.0 release, we've pushed v1.2.1 to address critical context limits when auditing large codebases. This update ensures that the pre-pass phase remains stable even as project complexity grows.

### Critical Fixes & Improvements
- **Chunked Pre-pass Sync Calls**: In PR #141, we fixed an issue where large sync calls would exceed the model's context window. The system now chunks these calls, allowing you to audit massive repositories without hitting token limits during the initial analysis phase.
- **Refined Release Pipeline**: v1.2.1 solidifies the changes introduced in PR #140, ensuring smoother CLI distribution and metadata consistency.

### Why this matters
If you are running Noxaudit on enterprise-scale repositories, you likely encountered `context_length_exceeded` errors during the pre-pass. This update mitigates those failures by intelligently managing how sync data is sent to the underlying model.

### Ship it
To get the latest improvements, update your local CLI:

`npm install -g @atriumn/noxaudit@latest`
