## Noxaudit v1.1.7 is Live

We have just released v1.1.7 of Noxaudit, focusing on improving the accuracy of findings when auditing codebases with incomplete test structures.

### Notable Changes

- **Inject finding when no test files exist ([#130](https://github.com/atriumn/noxaudit/pull/130))**: Previously, a project lacking test files might bypass certain checks silently. Noxaudit now explicitly injects a finding when no test files are detected. This ensures that missing test coverage is surfaced as a high-priority item rather than an empty result.
- **CI/CD Stabilization ([#127](https://github.com/atriumn/noxaudit/pull/127))**: Updates to our release automation to ensure consistent delivery following the v1.1.6 milestone.

### Implementation Notes

For teams running Noxaudit in CI/CD pipelines, v1.1.7 will help prevent "silent passes" on repositories that lack a testing suite. If your codebase is currently undergoing a migration and is intentionally missing test files, you may see a new finding injected into your audit reports.

To update to the latest version, run:
- `npm install -g noxaudit@latest` or update your project's devDependencies.

View the full release notes on [GitHub](https://github.com/atriumn/noxaudit/releases/tag/v1.1.7).
