## Noxaudit v1.1.5 is now available

This release focuses on optimizing the core audit engine and expanding CLI capabilities for automated environments. Following the v1.1.4 release, we have addressed several edge cases in dependency resolution and improved the developer experience for large-scale monorepos.

### Key Improvements
- **Faster Monorepo Parsing**: Optimized the AST traversal logic, reducing memory overhead by 15% for projects with over 500 modules (PR #284).
- **New CLI Flag**: Added `--json-summary` to provide a machine-readable audit report without the full trace log, ideal for custom dashboard ingestion.
- **Path Resolution Fix**: Resolved an issue where `--ignore-path` failed to respect nested `.gitignore` rules in certain recursive directory structures (PR #289).
- **Enhanced Error Codes**: Exit codes have been refined to distinguish between security vulnerabilities (exit 1) and configuration errors (exit 2).

### Breaking Changes
- The `--fail-on-warn` flag is now deprecated in favor of `--severity-level`. If your CI pipeline uses `--fail-on-warn`, please update your scripts to `--severity-level=medium` to maintain existing behavior.

### How to Update
Pull the latest version directly from the registry:

- `npm install -g noxaudit@1.1.5`
- Or update your `package.json` and run your install command.

View the full changelog and PR details here: https://github.com/atriumn/noxaudit/releases/tag/v1.1.5
