Noxaudit v1.1.2 is now available. This release focuses on refining the developer experience and clarifying our core messaging to better align with how you use the tool.

### What’s New

- **Optional Scheduling:** We have reframed scheduling as an optional feature rather than the default path ([#108](https://github.com/atriumn/noxaudit/pull/108)). Noxaudit now integrates more naturally into varied CI/CD environments where you may prefer manual triggers or custom hooks over a forced schedule.
- **Branding Refresh:** We have removed "nightly" branding from the tagline and core documentation ([#106](https://github.com/atriumn/noxaudit/pull/106)). This change reflects the tool's utility for on-demand security auditing at any time, moving away from the implication that it only runs as a background batch process.
- **Consolidated Fixes:** This update incorporates stability improvements and chore updates from the v1.1.1 release cycle.

### Why it matters
By making scheduling optional and removing the "nightly" label, we are ensuring Noxaudit remains an unopinionated part of your security stack. Whether you run it on every commit or as a standalone check, the tool now treats these workflows as equally first-class.
